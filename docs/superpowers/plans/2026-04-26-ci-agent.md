# Dogfood-3 CI Agent — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a CI agent (xtask recipe + script) that pulls an image via minibox, runs the
test suite inside the container with bind-mounted source, streams stdout to parse results,
and cleans up. Exercises the full ephemeral container lifecycle and validates minibox as a
real CI backend.

**Architecture:** No daemon changes. The agent is a `cargo xtask ci-container` command that
uses `minibox_core::client::DaemonClient` to drive the daemon programmatically. It exercises
`ephemeral: true` streaming, bind mounts, and container cleanup. Optionally wrappable as a
`just` recipe.

**Tech Stack:** Rust (`xtask` crate), `minibox-core` (client + protocol), `tokio`,
`cargo nextest`, Nushell (just recipe).

---

## Causal Chain

```text
T1: CI image definition                     (prereq for container runs)
  └─► T2: xtask ci-container command         (core orchestrator)
        └─► T3: Test result parsing           (exit code + output)
              └─► T4: Cleanup logic            (rm container on exit)
                    └─► T5: just recipe         (ergonomic entry point)
                          └─► T6: Tests + commit
```

**Note:** T1 can reuse the sandbox image from Dogfood-2 if already built, or define a
separate Rust-only image. T2-T4 are sequential. T5 is independent of T4.

## Task 2: xtask ci-container command

**Files:**

- Create: `crates/xtask/src/ci_container.rs`
- Modify: `crates/xtask/src/main.rs`

**Change:** Add `cargo xtask ci-container` that:
1. Connects to daemon via `DaemonClient`
2. Ensures `minibox-ci:latest` is available (pull or verify)
3. Sends `Run` with `ephemeral: true`, bind mount `./:/src:ro`, and `cargo nextest run`
4. Streams output line-by-line to stdout
5. Returns the container's exit code as the process exit code

- [ ] **Step 1: Add minibox-core dependency to xtask**

  ```toml
  # In crates/xtask/Cargo.toml
  [dependencies]
  minibox-core = { path = "../minibox-core" }
  tokio = { version = "1", features = ["full"] }
  ```

- [ ] **Step 2: Implement ci_container module**

  ```rust
  use anyhow::{Context, Result};
  use minibox_core::client::{DaemonClient, default_socket_path};
  use minibox_core::protocol::{DaemonRequest, DaemonResponse};

  pub async fn run(test_filter: Option<&str>) -> Result<i32> {
      let socket = default_socket_path();
      let client = DaemonClient::connect(&socket)
          .await
          .context("ci-container: connect to daemon")?;

      let command = match test_filter {
          Some(f) => vec![
              "cargo".into(), "nextest".into(), "run".into(),
              "--workspace".into(), "-E".into(), f.into(),
          ],
          None => vec![
              "cargo".into(), "nextest".into(), "run".into(),
              "--workspace".into(),
          ],
      };

      let request = DaemonRequest::Run {
          image: "minibox-ci".into(),
          tag: Some("latest".into()),
          command,
          memory_limit_bytes: Some(2 * 1024 * 1024 * 1024), // 2GB
          cpu_weight: None,
          ephemeral: true,
          network: None,
          env: vec![],
          mounts: vec![BindMount {
              source: std::env::current_dir()?,
              target: "/src".into(),
              readonly: true,
          }],
          privileged: false,
          name: Some("ci-runner".into()),
          tty: false,
      };

      // Send request, stream output, return exit code
      // ... (stream loop identical to mbx run)
  }
  ```

- [ ] **Step 3: Wire into xtask main**

  Add `CiContainer` variant to the xtask subcommand enum with optional `--filter` arg.
