Environment Variable Flow

Traces how user-supplied env vars travel from the CLI or mbxctl HTTP API all the way to execve(2) inside the container process.

Why execve not execvp

execvp inherits the parent process environment (the daemon's env), which leaks host secrets into containers. execve takes an explicit envp argument, giving the container exactly the vars listed in ContainerSpawnConfig::env — no more, no less.

Defaults (PATH, TERM) are prepended in handler.rs before the user vars so they can be overridden by the caller (e.g. env: ["PATH=/custom/bin"] replaces the default).
