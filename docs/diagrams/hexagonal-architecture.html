Hexagonal Architecture

Minibox uses hexagonal (ports and adapters) architecture. The domain ports (traits) in minibox-core/src/domain.rs define the interfaces. The application core in daemonbox depends only on those traits — it never imports a concrete adapter. Composition roots (miniboxd/main.rs for Linux, macbox::start() for macOS, winbox::start() for Windows) are the only place where concrete adapters are wired to the application core.

This means the daemon logic (handler, state, server) is fully testable with mock adapters and has zero platform-specific code.
